Multinomial processing tree (MPT) models are flexible tools to model categorical responses in terms of cognitive processes generating them. They have been used in many fields of psychology (for reviews, see  Batchelder & Riefer, 1999; Erdfelder et al., 2009; Hütter & Klauer, 2016 ) and continue to attract research attention both in terms of substantive applications as well as in terms of methodological extensions. One area that has recently seen considerable progress is extending the model class to incorporate not only the categorical responses, typically available as response frequencies, but to also fit response latencies ( Hartmann, Johannsen, & Klauer, in press; Heck & Erdfelder, 2016, 2020; Heck, Erdfelder, & Kieslich, 2018; Klauer & Kellen, 2018 ). Here, we focus on the approach and model class proposed by  Klauer and Kellen (2018) , termed RT-MPT models, which enables one to estimate completion times for each process and process-outcome along with a residual encoding and motor time.

One limitation of this approach is that none of the modelled cognitive processes is allowed to occur more than once with the same outcome along a given processing path of the underlying MPT model. Yet, such process repetitions occur, perhaps not often, but regularly in MPT modelling.  Batchelder and Riefer ’s ( 1980, 1986 ) prominent  pair-clustering  model is a point in case. This model applies to a  free-recall  task. Participants first study a list consisting of two types of words, members of pairs (two words that belong to one cluster) and singletons (single words belonging to no cluster;  Batchelder & Riefer, 1999 ). The repeated process here is the one for storing and retrieving of singletons. In general, the problem with repeated processes is likely to arise whenever multiple stimuli must be processed to reach a response category of the model.

The purpose of this short paper is to extend the RT-MPT approach to accommodate underlying MPT models with process repetitions along one or more given paths. We present the mathematical and statistical development necessary for doing so, validate the resulting algorithm for fitting RT-MPT models with repeated processes, and present an empirical application based on published data. The extension is available in a user-friendly form in a new version of  Hartmann et al. ’s ( in press ) R package  rtmpt  for fitting RT-MPT models.

As a running example, we will consider a two-high-threshold (2HTM) model of the two-alternative forced choice (2AFC) task, a standard paradigm in studying recognition memory ( Kellen & Klauer, 2018 ) . In this task, participants first learn a set of words (study phase), which they are to remember. In each trial of the subsequent test phase, they see two words horizontally aligned, one of them previously studied, the other new. Participants’ task is to decide whether the left word is old ( “old-new” ) or the right word ( “new-old” ).

In the 2AFC task, there are two types of test stimuli —  “old-new”  and  “new-old” . The 2HTM model for this task consists of two trees, one for each type of stimulus, and two response categories per tree: the  “old-new” -response and  “new-old” -response (see  Fig. 1 ). The MPT model class was recently extended by  Klauer and Kellen (2018)  to model process-completion times (short  process times ) and encoding plus motor execution times (short  motor times ) along with the process probabilities of traditional MPTs. It models response frequencies and response latencies jointly using a mixture distribution. Each processing path from root to a response category in the tree representation of the model has its own mixture component and each component consists of the distribution of the sum of one or more exponentially distributed process times and a normally distributed motor time truncated from below at zero, which leads to a modified  (hypo) exGaussian  distribution (see  Hartmann et al., in press; Klauer & Kellen, 2018 ). The mixture weights are the  path probabilities  (product of all probabilities lying on the path).

The problem of repeated processes arises because of the multiple exponentially distributed process times that are encountered along one processing path of the processing trees. For each process time there is an associated exponential rate parameter, one for each of the two outcomes of the process, referred to as  process-rate  parameter in the following.

Extending the RT-MPT model class to accommodate repeated processes basically amounts to replacing the original expression for the pathwise response-time distribution of Eq.  (1)  by the generalized response-time distribution of Eq.  (7) . We implemented this change in the MCMC algorithm for estimating RT-MPT models with up to five occurrences of the same process-outcome pair along a given path. For reasons of numerical stability, we recommend the use of the algorithm with no more than four such occurrences. 
                         2 
                       
                      
                         2 
                         The reason is that we also use these new expressions to better deal with numerical problems in the calculation of the log-likelihood that arise when two process rates from different processes along a given path accidentally assume values close to each other in the course of the MCMC estimation. If numerical problems are encountered going back to accidentally similar values, we now compute the likelihood at that point by considering these values identical and using the above formulae for repeated process rates. This temporarily increases the number of repetitions for one data point and MCMC iteration, which is only possible if the maximum number of repeated occurrences has not already been used up by systematic repetitions in the underlying model itself. 
                       We are not aware of any MPT model in the literature with this many repetitions.

The algorithm fits a hierarchical Bayesian version of the model in which person-level parameters for the traditional process-probability parameters, processing rates, mean motor times and variances are estimated for each person. These are, however, constrained by group-level distributions through which group means for the person-level parameters as well as their variances and covariances across persons are also estimated.

We subjected the extended Bayesian algorithm to a validation study, using the  simulation-based calibration  (SBC) approach proposed by  Talts et al. (2018) .  Talts et al.  argue that conventional methods like parameter-recovery studies are not appropriate for validating a Bayesian algorithm. For example, biases in parameter recovery do not imply that the algorithm is not valid; conversely, if no evidence for biases is found in a parameter-recovery study, it does not follow that the algorithm is valid. The SBC procedure, in turn, allows the conclusion that an algorithm is not valid if there are biases.

We applied the model to an existing dataset by  Province and Rouder (2012)  for which a standard 2HTM model applied to a 2AFC recognition-memory task results in a model with repeated processes.  Province and Rouder (2012)  argued that their data are compatible with a discrete model such as the 2HTM. The present analyses underline this conclusion by explicitly fitting a discrete-state model accounting for responses and response latencies simultaneously. Departing from Province and Rouder (2012), we found, however, that process-completion times for successful detection attempts decrease with increasing strength of old words. The application serves to illustrate the usefulness of the RT-MPT approach and its extension for repeated processes.

The extension has been included in the  rtmpt  package in R and enables one to fit models with process-outcome repetitions in one or more paths of the underlying MPT model. A limitation of the current implementation is that for each process-outcome pair and path, the number of such repetitions is limited to a maximum of five. Although MPT models with repeated processes occur regularly, especially when multiple stimuli have to be processed for a response, we are not aware of any model in the literature with five or more process repetitions. The methods explained here can relatively straightforwardly be scaled up to deal with even higher numbers of repetitions, if the need to do so should arise.
