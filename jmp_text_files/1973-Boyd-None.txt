M is a mathematical structure such as a  set or a Cartesian product. An extension of P61ya's theorem is proved which allows the number of M-trees for a given number of  nodes to be counted. The special case of componential trees is investigated. Here M is a Cartesian product of 0's and 1's. A  componential analysis is a componential tree of depth 1, that is, there is no hierarchy. We prove that for any componential  tree there exists a componential analysis which makes the same predictions on a triad test of judged similarity. A brief  empirical example is given, in which a componential tree is applied as a model of a semantic domain.  1. INTRODUCTION  Trees, or hierarchical structures, are ubiquitous. In nature, we find real trees and other structures such as river systems  and veins that have the branching structure of real trees. On a more abstract level, the Linnean classification of living  things is a classic example of a tree structure. Trees crop up in studies of cognitive structure ranging from the "tote  hierarchies" of Miller et al. (1960) to the "phrase markers" of Chomsky (1957). Family trees of the type found in the Bible  where Adam begat sons who begat other sons are trees. A difficulty arises if daughters are counted in addition to sons, for  then one's distant descendants may marry, closing a loop in a very untreelike manner. On the other hand, some relationship  must exist between two people before they can be said to be in the same family tree.  These two considerations motivate the graph theoretic definition of trees. NONESSENTIALNESS OF THE HIERARCHY  It turns out that componential trees with the associated process model do a very good job of predicting some triad choice  data. Can we take this as evidence that hierarchical structure and processing is necessary? In particular, can a componential  analysis make the same predictions as a componential tree?  A componential analysis is defined as a function from an M-structure into the set of points, where the M-structure is of the  form jZn{0, 1} for some n. In other words, a componential analysis assigns a word to each n-tuple of 0's and 1's for some n.  This is a rather restricted form of definition for componential analysis, since we are allowing only two values on each  dimension. However, it turns out that even this strong requirement allows our result.  A componential analysis is thus a special case of a componential tree. Thus the process model is defined. It simply computes  the city block distances between points and makes its decisions on that basis, with no hierarchical processing. We can now  state our question more precisely. For any componential tree, does there always exist a componential analysis which makes the  same predictions on the triad test? The answer is "yes," as we will show.  Before we proceee with the construction let us first note that although componential trees are not needed to predict  similarity experiments choices, it is nevertheless true that they might be better models for the structure of semantic domains  than are componential analyses. First, componential analyses will have, in general, very many "holes," that is, tuples of  feature values for which no lexical item exists. This will bethe case even when the componential tree from which it was  derived had no holes at all. Thus the features will seem rather arbitrary and artificial. The very large number of dimensions  needed for the componential analysis is also very unappealing.  There are also experimental reasons for preferring componential trees to componential analyses. Just because a componential  analysis predicts the same choices as a subject makes does not mean that it processes the items in the same way. In fact we  can put a simple model on componential tree processing which predicts response times to various kinds of triads (i.e., various  levels at which the choice is made) and these predictions have been confirmed (Wexler, 1970b). The componential analysis,  however, makes the wrong response time predictions. AN EMPIRICAL EXAMPLE  In this section we briefly present a componential tree which does well in predicting from a triad experiment on a set of  "have" words. The words were originally studied linguistically by Bendix (1966). For details of the data analysis, see Wexler  (1970b). For other examples of M-trees applied to these words (see Wexler, 1970a).  Here we will list the structure equations with only partial discussion. The structures are:  S(have) = 0, 0, 0,  S(keep) = 0, 0, 1,  S(get) = 0, 1, 000,  S(take) = 0, 1, 001,  S(borrow) = 0, 1, 010,  S(find) = 0, 1, 100,  S(give) = 1, 0, 0,  S(lend) = 1, 0, 1,  S(lose) = 1, l.  The structural equations were used to make predictions for an experiment in which 24 native speakers of English were asked to  pick the "word most different in meaning from the other two" when presented with three words. Thus, in (give, get, take) most  subjects selected give as most different in meaning, as the equations, together with the process model, predict. A summary  statistic yields a correlation of r = 0.98 between theory and data. That is, the model makes choices which are similar to  those that subjects make.  TREES WITH STRUCTURE 145  The first "cut," that is, the first element in the structure sequence represents whether or not the subject of the verb has  the object of the verb after the act has been performed (or after a certain time t). The verb is marked 0 if the subject has  the object and 1 if the subject does not have the object. Thus get is marked 0 because if, for example, "John gets the ball,"  then after the action John has the ball. The same is true for have, keep, take, borrow and find. However, if "John gives the  ball (to somebody)," then after the action, John doesn't have the ball. Thus, the first element in the sequence is l for give.  This is also true for lend and lose.  The second level in the sequence is also one-dimensional. Note that the interpretation of a value is different depending on  the preceding values in the sequence. Consider first the cases where the first value is 0. Then the second value is 0 if the  subject had the object before the action and 1 if he didn't have the object before the action. Thus have and keep are marked 0  and get, take, borrow and find are marked 1. If, on the other hand, the first value was 1, then the second value is 0 if the  action was "nonaccidental" and 1 if the action was "accidental." Thus, give and lend have the value 0 and lose has the value 1.  The only level which has more than 1 dimension is the third level for the verbs get, take, borrow and find, which has three  dimensions.
