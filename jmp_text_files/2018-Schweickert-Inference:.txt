Analysis of response time has a long history, (e.g.,  Helmholtz, 1883 ) as does analysis of response accuracy (e.g.,  Ebbinghaus, 1885/1913 ). Most analyses focus on one variable, not from belief that the other is unimportant but from lack of techniques for analyzing them in a unified way. A paper often reports an Analysis of Variance on response time, followed by another Analysis of Variance on proportion of correct responses, followed by discussion of where the separate analyses agree and disagree. An exception is work on certain mental processes, decision and search in particular, for which well-developed theories explain the duration of the process and the outcome of the process in a unified way (see, e.g.,  Ratcliff, 1978; Smith, 2000; Vickers, 1979 ). Theories about a single process often assume the other processes required in a task simply precede or follow the single process, but for many tasks such a serial arrangement is an oversimplification. There is a need for models that allow complex arrangement of individual processes yet treat response time and response class in a unified way. Multinomial Processing Trees are a promising foundation, already well established as models of response accuracy.

Multinomial Processing Trees are successful models in many realms, including perception, memory, and social cognition. For reviews see  Batchelder and Reifer (1999) ,  Erdfelder et al. (2009) , and  Hütter and Klauer (2016) . Modeling is usually of probabilities that responses fall into various classes, such as correct and incorrect. A few investigators, including  Link (1982) ,  Hu (2001)  and, recently,  Heck and Erdfelder (2016, 2017)  and  Klauer and Kellen (2018)  have extended Multinomial Processing Trees to response time. A potentially informative feature of such models is that they assume a person can perform a task in more than one way, so they are capable of explaining nonmonotonic relations between response time and accuracy  (Schweickert & Zheng, 2018a) . Many well developed techniques for response time analysis would be useful, if modified to apply to Multinomial Processing Trees. Here we present some such modifications, based on selectively influencing processes with experimental factors.

An experimental factor selectively influences a process if changing the level of the factor changes parameters unique to that process, leaving all else invariant.  Sternberg (1969)  pioneered selective influence for response times. It is now often used for response probability in Multinomial Processing Tree (MPT) models. For example, in an immediate recall experiment,  Chechile (1977)  found that changing the phonological similarity of items changed a storage parameter in an MPT, leaving other parameters invariant. As  Batchelder and Alexander (2013, p. 1209)  say, “In almost all of the articles proposing a new MPT model are selective influence studies designed to validate the interpretation of the parameter estimates”. For a survey of selective influence for response probabilities, see  Schweickert, Fisher, and Sung (2012) . For a general conception, see  Dzhafarov (2003) .

In a Multinomial Processing Tree ( Fig. 1 ), a vertex represents a process such as memory retrieval.  Schweickert and Chen (2008)  and  Schweickert and Xi (2011)  developed response probability tests of whether two factors selectively influence two different processes, each represented by a different vertex in an arbitrary Multinomial Processing Tree. Here we extend the tests to response times and other measures for the case in which responses fall into two classes, e.g., correct and incorrect. The tests are variations of established tests for response time (e.g.,  Dzhafarov, Schweickert, & Sung, 2004; Houpt, Blaha, McIntire, Havig, & Townsend, 2014; Roberts & Sternberg, 1993; Schweickert, 1978; Sternberg, 1969; Townsend & Nozawa, 1995 ). We investigate the feasibility of the tests with simulations.

A two class Multinomial Processing Tree is one in which responses fall into two classes. Two factors selectively influencing different vertices in such a Multinomial Processing Tree are very informative. The MPT is equivalent, as we will explain, to one of two relatively simple trees, so the investigator need only consider these two. If the two vertices are in order in the MPT, the order can sometimes be determined from the data. Notably, if the tests fail no two class Multinomial Processing Tree is possible in which the two factors selectively influence two different vertices. The investigator can quickly learn another type of model is needed.

If there are more than two response classes, every class can be tested separately. It is necessary that the separate tests are satisfied for every response class if a single MPT with a terminal vertex for each response class is able to account for the data, with each of the two factors selectively influencing a different vertex  (Schweickert & Zheng, 2017) . We emphasize that if the tests fail, an MPT may account for the data, but not an MPT in which each of the two factors selectively influences a different vertex.

A Multinomial Processing Tree ( Fig. 1 ) consists of points, called  vertices , joined by lines, called  arcs . Our introduction here is casual, for a more formal description see  Purdy and Batchelder (2009) . Each vertex represents a mental process, such as perception or memory retrieval. An arc descending from a vertex represents an outcome of the process, such as successful memory retrieval. When an event such as stimulus presentation occurs, the first process begins. In the figure this process is represented by the vertex at the top of the tree, called the  source . On a particular trial, a single outcome of the first process occurs, the arc representing the outcome is traversed, and the vertex at the end of the arc is reached. This vertex represents a further process, which is executed. Such steps continue until a vertex is reached that has no descending arcs. At such a  terminal vertex  a response is made. The steps form a path from the source to the terminal vertex. (Some authors say a vertex represents a state, and an edge or entire path represents a process. We do not.) Responses fall into various classes, such as correct and incorrect. A particular terminal vertex is associated with only one class of response.

Each terminal vertex has a probability of being reached on a path from the source. The probability a response falls into a particular class is the sum of the probabilities of the terminal vertices associated with the class.

Some processing trees include a measure in addition to probability. In a decision tree, each arc has a probability of being selected and a gain or loss is obtained. The net gain of a path is the sum of the gains and losses associated with the arcs on the path. In another example,  Rosenbaum (1980)  proposed a processing tree for the task of reaching straight ahead to a stimulus a certain distance away. At each vertex, an outcome is selected with a certain probability and the outcome adds or subtracts an amount to the distance goal. The distance reached equals the sum of the amounts in the distance goal.

Some examples incorporate time.  Rosenbaum, Kenny, and Derr (1983)  proposed a processing tree for a movement sequence, such as pressing keys on a piano. A vertex leads with a certain probability to the correct next vertex, or with a different probability to an incorrect next vertex. Executing the process represented by a vertex requires a certain amount of time. The time between two successive movements is the sum of the times required by the vertices on the path from one movement to the other.

Similarly, in  Hu ’s  (2001)  model when processing is carried out at a vertex, each arc descending from the vertex has a probability of being selected and takes time. Using the assumption that probabilities on a path multiply and times on a path add, Hu derived equations for response probabilities and expected response times for arbitrary processing trees.

In the model of  Heck and Erdfelder (2016, 2017) , the probability of a path is the product of the probabilities on the arcs on the path. Times are not associated with individual arcs, but when a terminal vertex is reached, a response time associated with the terminal vertex is produced. Heck and Erdfelder decompose the response time distribution into a mixture of the times required to reach each terminal vertex. Response times are analyzed in a distribution free way by estimating their density functions.

A paper by  Klauer and Kellen (2018)  assumes as  Hu (2001)  did that the probability of a path is the product of the probabilities of the arcs on it and the time for a path is the sum of the times of the arcs on it. An exponential distribution is assumed for the time of each arc, except that arcs whose ending vertex was terminal have truncated normal distributions. Data are analyzed with Bayesian hierarchical analysis.

An investigator constructing an MPT model confronts an initial difficulty. What processes should be included and how should they be arranged? The same difficulty is faced in modeling response times—if one does not know what the processes are, how can one account for the time? For response times, the need to know the process arrangement in advance was obviated by  Sternberg ’s ( 1969 ) Additive Factor Method. If all processes are in series, and each of two factors selectively influences a different process, then the combined effect of the two factors on response time will be the sum of their individual effects. This can be tested for mean response times with an Analysis of Variance test for an interaction.

For response time tests of process arrangement, Cumulative Distribution Functions are often analyzed. Developments include those of  Ashby and Townsend (1980)  for serial processes,  Roberts and Sternberg (1993)  for processes in series and mixture distributions,  Townsend and Nozawa (1995)  for processes in series or parallel,  Schweickert, Giorgini, and Dzhafarov (2000)  for processes in task networks, and  Heck and Erdfelder (2016, 2017)  for Multinomial Processing Trees.

For response probability tests of process arrangement in Multinomial Processing Trees,  Schweickert and Chen (2008)  employed selective influence. Consider an experiment with two factors. Changing the level of a factor may change parameter values on arcs in several locations in the MPT. A factor  selectively influences  a vertex in an MPT if changing the level of the factor changes parameters on arcs descending from that vertex, leaving all else invariant. (We sometimes say the factor selectively influences the process represented by the vertex.) Suppose each of the two factors selectively influences a different vertex in an arbitrary two class MPT. Two MPTs are  equivalent  if they make exactly the same predictions for all combinations of levels of the factors.  Schweickert and Chen (2008)  showed that if response probabilities alone are measured, the arbitrary two class MPT is equivalent to one of the two relatively simple MPTs in  Figs. 1 and 2 . (The term equivalent is used in the literature in several ways. Here it is defined with respect to the two particular factors and their two particular sets of levels.) The process arrangement need not be known in advance, because tests can be conducted to determine whether one, the other, or neither MTP accounts for data from a particular experiment. Response rates can be analyzed, instead of response probabilities  (Schweickert & Xi, 2011) .

It has recently been shown that even if a supplementary measure such as response time is included in a two class Multinomial Processing Tree, an investigator need only consider the two trees in  Figs. 1 and 2  
                      (Schweickert & Zheng, 2017, 2018a) . Consider an arbitrary two class Multinomial Processing Tree with arc probabilities and times. Suppose the probability of a path is the product of the probabilities on the arcs on the path, and the time for a path is the sum of the times on the arcs on the path. This assumption, made by  Hu (2001) , is supported in a variety of psychological tasks (e.g.,  Schweickert, 1985; Schweickert, Fortin, Xi, & Viau-Quesnel, 2014 ). If each of two factors selectively influences a different vertex, an arbitrary two class MPT is equivalent to one of the MPTs in  Figs. 1 and 2 , provided the probability of a correct response is strictly between 0 and 1 for every combination of factor levels.

Moreover, the same MPT need not be used on every trial, or by every subject. Suppose response probabilities and times are aggregated over a mixture of two class MPTs and each factor selectively influences a single vertex in each MPT of the mixture. If in each MPT those vertices are unordered, then the MPT in  Fig. 1  will account for the data. If the selectively influenced vertices are ordered, with the vertex selectively influenced by a given factor always preceding that selectively influenced by the other, then the MPT in  Fig. 2  will account for the data. (Selectively influenced vertices unordered in some MPTs of a mixture and ordered in others have unknown effects.) For details, see  Schweickert and Zheng (2018a) .

In this paper we develop tests of response probabilities and response times, to determine whether one, the other, or neither of the two trees in  Figs. 1 and 2  is possible for data from a factorial experiment. Failure of the tests is instructive. Failure does not imply that an MPT model cannot account for the data. But if neither of the MPTs in  Fig. 1  or  Fig. 2  fits the data from a two-factor experiment, then under reasonable assumptions no two class MPT exists in which the two factors selectively influence two different vertices. The investigator can turn to another type of model, perhaps an MPT model in which a factor influences more than one vertex.

Before going on, we summarize restrictions imposed in this paper and consider why they do not greatly limit the usefulness of the results. First, an MPT is assumed to have a single source. Some MPT models have more than one. For example, the one-high threshold model  (Swets, 1961) , has a source vertex for targets and another for lures. But usually only one source is relevant on a given trial and the investigator can test the data separately for each source. Second, an MPT is assumed to have terminal vertices for only two response classes. But if there are more than two response classes, trials on which a particular response class was made can be tested separately, ignoring the other classes. If two factors selectively influence two vertices in an MPT with more than two response classes, it is necessary that the tests developed here are passed for each response class separately. Third, the times associated with arcs are random variables, but the probabilities associated with arcs are fixed. For one particular test, Eq.  (19) , times associated with two arcs are assumed to be independent. These are strong assumptions, but somewhat weaker assumptions are in  Appendix A  and less stringent assumptions are being developed  (Schweickert & Zheng, 2018b) .

We need consider only two trees because there are only two possible arrangements of two vertices selectively influenced by two factors. Suppose there is a path from one of the vertices to the other, the path made of descending arcs, and the path includes an arc whose parameters change when the level of one of the factors changes. Then the two vertices are  ordered . If there is no such path, then the vertices are  unordered . Sometimes to be more precise, we use the terms  ordered by the factors  and  not ordered by the factors  for  ordered  and  unordered , respectively. We now consider the two possible trees in turn.

Tests of Eqs.  (9) and (10)  are tests of whether the factors have additive effects on the probability of a correct response and also have additive effects on correct response time multiplied by the probability of a correct response. These tests are analogous to the Additive Factor Method test  (Sternberg, 1969)  of whether factors have additive effects on response time means, to test whether factors selectively influence processes in series. Tests of additive versus interactive effects of factors on response time have been in continual use for a long time because they are very informative about mental process arrangement; see  Reimer, Strobach, and Schubert (2016)  and  Sung and Gordon (2018)  for recent examples.

We note that tests developed here have a different orientation from a common way of testing an MPT model by estimating its parameters and evaluating a goodness of fit statistic. Tests here are of necessary and sufficient conditions for one or the other of the Standard Trees to account for the data. The tests, if passed, imply that suitable arc probability and time parameter values exist, but they do not directly produce estimates of parameter values or of a goodness of fit statistic, which, of course, are useful in their own right.

An experimental factor can change mean durations without producing stochastic dominance; the assumption of stochastic dominance is rather strong. It was proposed for response times by  Sternberg (1973) , who called it the  Long RT Property  and found evidence for it in memory scanning experiments. There is by now considerable evidence that factors produce stochastic dominance for response times in a variety of experimental paradigms (see, e.g.,  Fific, Nosofsky, & Townsend, 2008; Schweickert et al., 2012; Sung, 2008; Townsend & Ashby, 1983; Townsend & Nozawa, 1995 ).

The stochastic dominance assumption was made and tested for response time in RT-extended MPT models by  Heck and Erdfelder (2016) , and results were generally favorable. In their response time data, in one case evidence supported the assumption for all four participants (target-detection stochastically dominated guessing-old). In another case, evidence supported the assumption for two of four participants (lure-detection stochastically dominated guessing-new) and was inconclusive for the other two.

We now modify the assumption that stochastic dominance is produced on arc times by incorporating arc probabilities. It is natural to consider the effect of a factor on the product of these quantities.

This concludes our development of tests. To see whether the tests are feasible and effective, simulations of the two standard trees were carried out. Results of the simulations are generally encouraging, but we will see that power is sometimes low for testing one of the conditions, Eq.  (12)  for arc  B  time differences.

Experiments with two factors were simulated. A level  i  of Factor  
                         Φ 
                       combined with a level  j  of Factor  
                         Ψ 
                       is called a  treatment . In each treatment 1000 trials were run. While this number of trials is larger than typical in an experiment, it is not unrealistic and such a sample size is occasionally used to examine response time distributions (e.g.,  Burbeck & Luce, 1982; Wandell, Ahumada, & Welsh, 1984 ). Although time and cost limit the number of trials, an experimenter can often manipulate factors so they produce sizeable effects. For example, in an experiment on immediate recall by  Vitevitch, Chan, and Roodenrys (2012) , probability of a correct response ranged from .20 to .85, with serial position having a large effect. (See  Schweickert & Han (2016) , for an MPT model of the probabilities.) In simulations here, 1000 trials were run in each treatment and factor levels were selected so they generally had substantial effects. Parameters and factor levels were not chosen to be ideal for demonstrating the effectiveness of the method nor to explore the range of possibilities. In some paradigms, such as serial recall, an error can be made by failing to respond, producing no response time. For simplicity, we considered such a situation and only correct response probabilities and times were simulated. Simulations were carried out in MATLAB and analyzed in R. We will see that simulated data from each standard tree produced results that support it, and the simulated data easily distinguished the two standard trees from each other.

Because the conclusion that a test is satisfied is acceptance of a null hypothesis, we estimate power. We will see in the results that with our sample size of 1000 trials, estimated power is high for most tests, but sometimes not for the test in Eq.  (12) . Although additional sample sizes could be explored, the sample size of 1000 is well situated for showing where power is high and where it is not.

Several estimators in the tests we propose have approximately normal distributions. To briefly consider them, first, the number of correct responses in a set of trials has a binomial distribution, which for a large number of trials has approximately a normal distribution. Second, products of expected values, such as  
                         π 
                      ( i,j ) 
                         
                            
                               τ 
                            
                            
                               k 
                            
                         
                      ( i,j ), are equivalent to expected values of particular random variables (see  Appendix A ) and are estimated as averages. The averages are approximately normally distributed by the Central Limit Theorem.

Consequently, some proposed tests are of linear combinations of estimators that have approximately normal distributions, so the linear combinations themselves have approximately normal distributions. When the assumption of approximately normal distributions is tenable, a suitable way to proceed for tests involving linear combinations of means is to use Analysis of Variance and conduct a priori tests of the linear combinations.

How good the approximation to a normal distribution is depends on the circumstances. Two features of our simulations boost the accuracy of the normal approximation. First, the arc probabilities are fixed numbers, simplifying expressions to be tested. Second, the number of trials in each condition is relatively large, 1000. However, some conditions to be tested involve complicated expressions for which a normal distribution may be only a rough approximation, making parametric tests difficult under general conditions.

Bootstrap confidence intervals do not require an assumption that the population distribution has a particular form, normal or otherwise. They are generally suitable for complicated expressions; see e.g.,  Efron and Tibshirani (1986) ;  Eidels, Houpt, Altieri, Pei, and Townsend (2011)  and  Van Zandt (2002) . We used bootstrapping to conduct statistical tests.

The basic idea is straightforward. One starts with an available sample of data. Many samples of the same size, called resamples, are formed from the sample by sampling from it with replacement. A value of the quantity under consideration is calculated from each resample. From the values of the quantity in all the resamples, one finds the upper and lower bounds such that the quantity is within the bounds in a proportion  
                         α 
                       of the resamples .  If the original sample were the population, the interval would be an estimate of a  
                         1 
                         − 
                         α 
                       confidence interval for the quantity. It turns out that although the original sample is not the population, under reasonable assumptions the interval is nonetheless a good estimate of a  
                         1 
                         − 
                         α 
                       confidence interval for the quantity; see, e.g.,  Efron and Tibshirani (1986) . We conclude a test is satisfied if the value it requires for a quantity is within a confidence interval.

Convenient R packages exist for response time interaction contrasts (e.g.,  Houpt et al., 2014 ) and for MPTs (e.g.,  Heck, Arnold, & Arnold, 2017; Singmann & Kellen, 2013 ). Some, such as the last, provide bootstrapping. But because they do not treat the functions needed here, new code was written.

Bootstrap confidence intervals were computed in R, using command boot in the R package boot. We have several interaction contrasts to test. A reasonable way to conduct the tests would be to obtain a set of bootstrap samples and use that set to obtain a bootstrap confidence interval for every contrast we consider. Using that one set of bootstrap samples would be efficient, but has the drawback of making the tests dependent on one another because they use the same set of samples. Because we were not sure how the tests would behave, we obtained a new set of bootstrap samples for every contrast we tested, although such new samples are not necessary. For every test, 10000 new bootstrap samples were taken from the 1000 generated trials for each treatment relevant to the test, both correct and incorrect. (In the terminology of the R boot package, the data are in the form of a dataframe, each row is an observation (a trial), the type of simulation is ordinary (the default value of sim), and the number of replicates was 10000, stratified by treatment).

Omnibus tests could be carried out, but when more than one contrast is possible, it is informative to examine results for each separately. Failure of a model’s prediction for one treatment but not for another might suggest a way to modify the model. Although an adjustment would ordinarily be made for the number of contrasts to be tested, we test each at the .05 level because that level is familiar and there would be little change in the conclusions if an adjustment were made.

Although power of a test could be estimated by bootstrapping, the computations required are greater than those required for conducting the test itself. The additional computations do not seem warranted for data from simulations here. Instead, we report approximate power based on assuming approximate normal distributions in tests for which that assumption is reasonable. These estimates, although sometimes rough, are more than adequate to see where power is high and where it is not.

Multinomial Processing Trees are used widely in psychology in terms of topics, but narrowly in terms of dependent variables, almost exclusively for response probability. Extensions to other measures have been developed by  Link (1982) ,  Hu (2001) ,  Rosenbaum (1980) ,  Rosenbaum et al. (1983) ,  Heck and Erdfelder (2016, 2017) , and  Klauer and Kellen (2018) . With developments so far, an investigator usually proposes a particular Multinomial Processing Tree at the outset, perhaps with a few variants. That tree and variants are then tested and compared with overall goodness of fit measures. If the tree is complicated, expressions it predicts for the dependent variables will be complicated also.

Here we describe a different approach. The investigator begins with data from an experiment with two factors, and tests whether there exists any two class Multinomial Processing Tree in which each factor selectively influences a different vertex. If there are more than two response classes, the data for each response class can be tested separately. It is necessary that the tests be satisfied for each response class separately for a Multinomial Processing Tree with more than two response classes to account for the data with each factor selectively influencing a different vertex. The tests do not require specification of a particular Multinomial Processing Tree and are distribution free. If the tests are not passed, the investigator learns that another type of model must be considered, perhaps a Multinomial Processing Tree in which each factor has influence in several places. If there are two response classes and the tests are passed, the factors selectively influence vertices in an MPT that is equivalent to one of two relatively simple Multinomial Processing Trees. The tests indicate which MPT applies.

For the two Multinomial Processing Trees, simulations show that for factors having substantial effects on response probability and time, the tests are diagnostic, with a number of trials that is large but feasible for experiments.

The tests are natural extensions of tests of interactions already widely used in response time analysis. Tests involving mean response times are analogous to the test of additivity used in the Additive Factor Method  (Sternberg, 1969) . Tests involving distribution functions are analogous to tests of cumulative distribution function contrasts  (Dzhafarov et al., 2004; Roberts & Sternberg, 1993; Townsend & Nozawa, 1995) .

Multinomial Processing Trees enjoy success as models of many phenomena, in psychology and other fields. Tests will be more stringent with response time or other measures included. It is likely that for some phenomena, we will find that a Multinomial Processing Tree model is not a complete description after all. But if we find a Multinomial Processing Tree model that accounts for both response probabilities and response times for a phenomenon, that model is quite firmly established.
