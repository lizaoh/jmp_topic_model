KUCK2  The Technological Institute, Northwestern University, Evanston, Illinois  In this report finite state grammars are used as representations for a variety of recognition procedures. The simplest  applications are for the recognition of objects. More complex grammars are described for systems with internal constraints. We  distinguish between simple systems containing an integrated set of grammatical rules, and complex systems of several disjoint  grammars in which the functions of recognition and prediction are separated. These disjoint systems have certain advantages.  As multi-purpose systems they can be used with considerable flexibility and in a variety of ways. Moreover, additions to  existing subsystems can be made with relative ease and new control grammars can be added without major reorganization of the  original processing system.  INTRODUCTION  Of the three sentence-generating grammars discussed by Chomsky (1959) only two have received major attention. The reasons for  this differential treatment are clear. For the study of natural language, only the Type 1 and 2 phrase-structure grammars  appear to be appropriate. On the other hand, Chomsky (1959) and Chomsky and Miller (1958) have demonstrated quite conclusively  the inappropriateness of the Type 3 grammars as a model for natural language.  In this paper, we will not be concerned with natural language but will restrict our attention to the recognition and  categorizing of objects. More specifically, we will discuss the design of recognition "processors" (Gorn, 1963) or devices  that can accept sensory inputs and decode them. We assume, first of all, that the design of a recognition device is formally  equivalent to the development of a suitable grammar. Furthermore, we assume that the Type 3 grammars are appropriate models  for a variety of perceptual phenomena and it is this class of phenomena with which we are concerned. By imposing a series of restrictions on these rewriting rules, one can construct either a Type  1, 2, or 3 grammar. The Type 1 grammars employ the smallest number of restrictions while the Type 3 grammars are most strongly  constrained.  For the Type 3 grammars, it is customary to define an initial starting state So and to use ,# as a marker symbol to indicate  the beginning and end of a string. Furthermore we will write our productions so that a sentence will be defined as the "string  of symbols emitted as the system moves from So to a first recurrence of So" (Chomsky, 1959, p. 150).  Our Type 3 grammars always contain the following. There will be an input alphabet of two elements, 0 and 1, and input strings  will consist of sequences of 0's and 1's. Our output alphabet will vary in size but will always be finite. Output alphabets  will be defined with the use of English letters, and output strings will consist of sequences of these letters. In addition,  we will use Greek letters to define the state of the system. Again, the number of possible states for any given grammatical  system will be finite.  In Chomsky's terminology (1959, p. 142), a Type 3 grammar is distinguished by the limits imposed on its productions. These are  restricted to the following two forms:  or  ai --* aka,  ai -->. a; , (2)  The first form of rewriting rule permits a given state description, ai , to be replaced by a single letter from the output  alphabet and a new state description. The second form of rewriting rule substitutes a single output or terminal symbol for a  state description.  There is one important respect in which our systems differ from those of Chomsky and Miller. Their interest was in the design  of systems that generate well-formed strings of symbols, in conformance with a set of grammatical rules. Thus, their systems  have no inputs and use a single terminal alphabet for representing outputs. Since our interest is in the design of processing  systems, we use both inputs and outputs and a pair of alphabets for their representation.  Our notation for productions or rewriting rules will make explicit reference to both input and output symbols. Productions  will be written as acg --* ilak a; . In this form, a9 and a; represent states. The first symbol, i, to the left of the slash  is always an input symbol, and the second, ak , is always an output symbol. Thus, this rule states that when a system in state  a9 receives an input i, the outcome is a transition to state a; and the production of the output symbol ax . Given a set of objects or events to be discriminated we will assume that they can be described by a set of  properties or defining attributes, each of which will take on a limited number of values. The defining attributes represent  sensory information and correspond to a string of input symbols. Each attribute would be represented by a single position in  the string and its value by the value found at that position in the string. For our purpose, we will limit each attribute to a  range of two values, which might be viewed as the presence or absence of some characteristic property or as the choice between  two possibilities. In general, an object will be defined by the values of several attributes so that each output will  correspond to a set of values for several elements in an input string. Thus, the problem for a recognizer will be to accept a  set of attribute values as input and to decode them. The result will be the recognition of the object displayed.  We need to make one further assumption about these input codes. We will not assume that the codes are optimal in any sense but  only that they are uniquely deconcatenable (Gorn, 1963). By this restriction, we mean that there will be no ambiguity about  when the recognition of a given object has been completed. In what follows, we will specify more precisely the meaning of this  assumption.  As an example of a recognition grammar, we will begin with a set of six output categories (a, e, u, b, c, d). By examining the set of codes in Table 1, we can define more precisely what we mean by uniquely deconcatenable. When any two  codes are compared beginning with each code's first element, it will never happen that one code is contained within another.  It is this property of the input codes that assures our ability to recognize without ambiguity each of the objects in the  output set.  We can also make some comments on the functional properties of these processing systems. Suppose we have an output with an  input code that is of length k. If we omit from consideration the special states that have to do with starting and stopping  the system, then we note a simple relationship between length of a code and the number of state transitions required for its  recognition: they are equal since each transition from one state to another effectively consumes an element from the input  string. Thus, in a certain sense, many of the predictions that we would make on the basis of information theory hold also for  these processing systems, provided that the input codes are optimal in the sense of Shannon and Weaver (1949) or Huffman  (1952). For example, while average length of an optimal code for equally likely categories will increase logarithmically with  the number of alternatives, so will the average number of state transitions necessary for recognition. Similarly, for symbols  of unequal probability of occurrence, we expect shorter codes for the more likely categories. The parallel statement that we  can make for a processor is that the more likely categories will be recognized with a minimum number of state transitions. To  the extent that state transitions represent  320 KRULEE AND KUCK  processing time, it would also follow that recognition time will be inversely related to the probability of occurrence of an  event or to the number of attributes necessary to define an object or category. Chomsky (1956) had originally pointed out that a finite state grammar can be used as a model for a finite state Markov  process and that these grammars are an equivalent representation for the state diagrams described by Shannon and Weaver  (1949). The simplest kind of memory is required for the recognition of a single letter in the output alphabet. In general, it is always possible to write a grammar which will take into account all constraints  up to and including those of the Nth order. If we examine the productions in Fig. 2, it becomes apparent that at least two different functions are being performed at  different times. Certain productions are primarily an aid to object recognition; they examine an element in the input string  and effectively record its value. Under some circumstances, these same productions will also produce an output. Certain other  productions perform primarily control functions with respect to either the second- or third-order predictive systems. For  example, when in state all the applicable productions control the functioning of two possible predictions. If the presence of  a "b" is confirmed, the system prepares to execute a third-order prediction that has also been confirmed, i.e., the presence  of the  326 KRULEE AND KUCK  sequence ab followed by e. If a "b" is not present, the system is released from the thirdorder prediction, now disconfirmed,  and control is transferred to the second-order system.  What we propose is that the functions of object recognition and of prediction/control be separated. This separation can be  accomplished with the design of two or more disjoint systems, each of which is a finite state grammar. One will function as a  recognizer and the other as a controller. These systems will be structurally independent although functionally interdependent.  Let us define more precisely each of these processing systems. The total system will make use of four alphabets, two of which  will be for communication between the total system and its environment, while the other two will be used for internal  communications between the two subsystems. For communication with the environment, the system will use the same input and  output alphabets as were previously defined. However, only Grammar R, the object recognizer, will have access to these  alphabets. It alone will process the inputs and produce the final outputs from the system. We will refer to these two  alphabets as I and O, respectively. Alphabet B will be used as a second output from Grammar R and as the only input to Grammar  P, the control system. Alphabet B will use the symbols Bi , B; , etc., in order to inform the controller about the state of  the object recognition system. Finally Alphabet C will be used as the output from Grammar P to Grammar R. This alphabet will  use only 8, and 81 in order to control the behavior of the object recognition system. On the whole, Grammar R will do the work  of input processing and will have no ability to make predictions. Grammar P will know about the inputs (i.e., the environment)  only indirectly through information from Grammar R. Grammar P will make its predictions and inform the object system of their  implications. A block diagram of these relationships between the two systems is described in Fig. 3. Only Grammar R contains the starting state So or provisions for returning to this state. Furthermore, we assume that  Grammar P will be found in state au when processing begins. In addition, only one machine functions at a time. Having introduced this distinction between simple and disjoint systems, there are certain questions that one might raise in  order to compare their functioning. Which class of systems processes information more rapidly? Assuming two systems with the  same predictive capacity, the simple grammar will always require less time in order to process its inputs. When two or more  disjoint grammars are employed, their functioning will always include all the processing time that would be required by the  simple grammar.
