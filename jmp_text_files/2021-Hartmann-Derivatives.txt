Another area where these results will be useful is in Bayesian estimation of these models requiring gradient information. Gradient information is required for the very widely used adaptive-rejection sampling algorithm ( Gilks & Wild, 1992 ) implemented in general-purpose software such as WinBUGS ( Lunn, Thomas, Best, & Spiegelhalter, 2000 ), and Jags ( Plummer, 2003 ) as well as for the highly efficient Hamiltonian algorithm for MCMC estimation ( Neal, 2011 ) used in STAN ( Carpenter et al., 2017 ). Both Jags and STAN have already been endowed with functions for the first-time passage density of the Wiener process ( Wabersich & Vandekerckhove, 2013, 2014 ). Gradient information on the CDF is required in MCMC estimation using the above sampling algorithms when dealing with censored or truncated data (e.g., when a response deadline is used,  Pleskac, Cesario, & Johnson, 2017; Ratcliff, 1980, 1988, 2006 ).

To make these newly derived partial derivatives available, we developed a software package within the programming language  R  ( R Core Team, 2020 ), called  WienR . This package provides all above mentioned partial derivatives and also the PDF and CDF of the first-passage time.

The package can be installed with the usual  R  command  install.packages("WienR") . For each derivative, gradient, PDF, and CDF there is a separate function (see  Table 1 ).

Since there are already  R  packages with some functions of the Wiener first-passage time on the comprehensive  R  archive network ( https://cran.r-project.org ) — the main repository for  R  packages — we decided not to follow the convention to name a PDF function with a leading “d” and a CDF function with a leading “p” followed by an abbreviation of the distribution name (e.g.,  dnorm  for the PDF of the normal distribution). Instead we named the PDF  WienerPDF  and the CDF  WienerCDF . The corresponding derivatives (for the 3-parameter Model) are called the same but with a leading “dt” (only for the PDF), “da”, “dv”, or “dw” for the derivatives with respect to the first-passage time, the upper barrier, the drift rate, or the relative starting point, respectively. For example, the derivative of the CDF with respect to the upper barrier is called  daWienerCDF . For the additional parameters in the 7-parameter diffusion model (see Section  7 ) the  R  functions have a leading “dt0”, “dsv”, “dsw”, or “dst0” for the derivatives with respect to the non-decision time, the inter-trial variability in the drift rate, the inter-trial variability in the relative starting point, and the inter-trial variability in the non-decision time, respectively. There is also the possibility to calculate the gradient of the PDF and CDF with the functions  gradWienerPDF  and  gradWienerCDF , respectively.

Every function has the same arguments, namely the first-passage time  t , the given  response , the upper barrier  a , the drift rate  v , the relative starting point  w , the optional non-decision time  t0 , the optional inter-trial variability of the drift rate  sv , the optional inter-trial variability of the relative starting point  sw , the optional inter-trial variability of the non-decision time  st0 , the optional  precision , the optional number of components in the infinite sums  K , and the optional number of threads  n.threads . If  precision  is provided, then the number of components necessary to ensure this precision is calculated and used. If it is not provided, either a default value of 1e-12 is used as  precision  or if  K  is provided then  K  is used as number of components. We also enabled  multithreading  in case many data points are used. For that purpose,  n.threads  can be set to  TRUE  to use all available threads of the computer or to a number. If this number exceeds the number of available threads of the computer then only the number of available threads is used. The functions are all vectorized; that is, a vector can be passed for each of the main arguments ( t, response, a, v, w ) as well as for the optional four diffusion parameters ( t0, sv, sw, st0 ).

Each derivative/gradient function returns the derivative/ gradient of the PDF/CDF and the PDF/CDF functions return the PDF/CDF value and the logarithm of that value.

The program decides automatically whether to use the small-time or the large-time representation of the corresponding function. This is done by comparing the necessary number of components to ensure the precision. If only  K  is provided, but not  precision , the hypothetical number of components for the default precision is calculated for both representations and the representation with the lower number of necessary components is used with the provided  K .

For efficiency reasons, we integrated  C++  code into the  R  package to do the calculations. In each function, after the arguments of the functions are checked, a  C++  program is called via a dynamic library. In this program almost everything is calculated on the logarithmic scale. This improves the accuracy and robustness of the calculations.

In this manuscript, we provided small-time and large-time representations for the derivatives of the PDF and CDF of the first-passage times of Wiener processes with constant drift rate. We derived expressions for the error incurred by truncating infinite series involved in the computation and expressions for the number of components in the series that need to be considered to reach a prespecified precision. As already mentioned, the derivatives are useful in various frequentist and Bayesian estimation algorithms that require gradient information.

A standard optimization routine (the BFGS algorithm) using our gradient information was always more precise, considering the best of five optimization runs from different initial values, than another standard routine, the simplex, that does not use gradient information. The former method always found larger values of the likelihood than the latter. Moreover, BFGS with our gradient information was almost always more precise in this sense than BFGS with gradient information determined numerically from the likelihood. In both comparisons, BFGS with our gradient information was fastest. Similar, if less pronounced, differences were found for an analogous comparison based on the 7-parameter diffusion model. For both the 4-parameter and the 7-parameter model, BFGS optimization based on our gradient functions led to considerable increases in the speed of fitting the models.

We extended the PDF, CDF, and derivative computations to the case of the 7-parameter diffusion model. Our results are implemented in a user-friendly way in a new  R  package,  WienR , which we subjected to a number of validity and consistency checks.

Considering the computation of the PDF and the CDF of the 3-parameter model, the checks for the two packages  RWiener  and  WienR  indicate that the two packages calculate the same underlying PDF and CDF. This is largely also true for the package  rtdists . For unknown reasons, the package  rtdists , however, produces incorrect CDF values for some of the data points as described in the supplemental materials. Similar results were found for the 7-parameter PDF and CDF computations in comparing  WienR  and  rtdists  ( RWiener  does not implement the 7-parameter model). Note that  RWiener  and  rtdists  provide additional functionality such as quantile functions as well as sampling functions. Note also that for higher levels of chosen precision (such as 1e-10 for the PDF and 1e-5 for the CDF), computing 100 PDF or CDF value for the 7-parameter model using  rtdists  requires many minutes, but only a few seconds using  WienR  (see supplemental materials for more detail). The PDF and CDF implementations in  WienR  thus perform substantially faster than those in  rtdists  for the 7-parameter model, but they are only a little faster than the implementations in  RWiener  and  rtdists  for the 3-parameter model.

The derivative functions in the new package  WienR  are quite close to the numerically calculated derivatives using the  numDeriv  package or a simple two-point approximation, both for the 3-parameter model as well as for the 7-parameter extension. These results raise our confidence in the correct implementation of the derivatives in  WienR .

For both models, the derivatives are computed faster than through the use of  numDeriv  and the simple approximation, and the current methods control the approximation error, whereas no error bound or error estimate is provided in these other methods. As illustrated above, use of the gradient information can improve speed and accuracy in frequentist model-fitting exercises, and we believe it to be similarly useful in implementing Bayesian MCMC methods relying on gradient information. Taken together, we have provided representations and implementations that we hope will prove useful for many mathematical psychologists working in the field.
